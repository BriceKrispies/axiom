Axiom — Asset Streaming on a Worker Pool

The engine skeleton boots first — this page is interactive immediately — then a pool of background Web Workers streams the assets. Each worker fetches and decodes one asset off the main thread; the main thread only runs the deterministic axiom-assets scheduler, handing queued jobs to whichever workers are idle. CPU-heavy decode never blocks the frame. Set the pool size with ?workers=N (default 3).
