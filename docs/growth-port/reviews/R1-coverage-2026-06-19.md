## 0. Bottom line

The registry is **broad but inflated**. The hydrology/atlas/streaming spine is genuinely
real (good code, real tests). But:

- **Two platform requirements name PHANTOM SITES** ‚Äî `SA-E4` and `SA-E5` both point at
  `genome::‚Ä¶` symbols that **do not exist** in `genome.rs`. The form‚Üísites mapping and the
  site-count guardrail are simply not where the registry says they are (and SA-E4's mapping
  is not implemented at all).
- **The entire async Sim-API surface (SA-0.2, SA-E1, SA-E2) is absent.** `SA-E1`/`SA-E2`
  aren't even in the registry; `SA-0.2` is `Deferred` with a note, not code. There is no
  `WorldGenJob`, no `start/poll/cancel/take`. Generation is a single blocking `Growth::generate`.
- **Climate is mislabelled DOWN, and ordered WRONG.** `OW-E7/E8/E9` are tagged `Scaffolded`
  but are actually fully-implemented stages **that run in `DEFAULT_GLOBE`** ‚Äî yet they run
  *after* the rivers, **contradicting the audit's required order** (OW-E8: "after elevation,
  **before** `triangle_values`"). So the registry under-claims the code while the code
  violates the spec. Both directions are wrong.
- **Moddability is a THIN CATEGORY bordering on fictional.** `MOD-defs` ("defs:: data-driven
  config") points at `defs.rs`, which is **17 lines**: one struct never constructed and one
  `const PERFORMANCE_GLOBE = DEFAULT_GLOBE` alias. No XML, no `DefDatabase`, no `manifest`,
  no biome loading, no pack order. The audit's central engineering bet (content-pack engine)
  is essentially unbuilt and over-claimed.
- **Persistence is entirely absent** (`OW-P1`, `GW-E7`): `Deferred`/missing, no serialize code.
- **Gameplay intents are dead wiring.** `GW-E10` `IntentRouter` is a unit struct with `new()`
  and **no route/handle methods**; `dig` bypasses it entirely (`Growth::dig` ‚Üí `apply_dig`
  directly). This matches `critical_bugs_audit.md #7` ("IntentRouter bypassed"). The registry
  hides this behind a generic `Scaffolded`.

## 5. Prioritized list of unaccounted / inflated requirements

**P0 ‚Äî false coverage (registry asserts a site that is wrong or empty):**
1. `SA-E4` ‚Äî phantom `genome::world_size mapping`; the form‚Üí`sim_voronoi_sites` map **does not exist**.
2. `SA-E5` ‚Äî guardrail attributed to `genome`; actually a subdiv cap in `topology.rs`; no `max_voronoi_sites` profile knob.
3. `MOD-defs` ‚Äî `defs.rs` is an alias + unused struct; "data-driven config" is fiction.
4. `PIPE-perf` ‚Äî `performance_globe` is a literal alias of `DEFAULT_GLOBE` (no fewer iterations).
5. `GW-E10` ‚Äî `IntentRouter` is an empty unit struct; dig bypasses it.

**P1 ‚Äî spec-violating despite "present":**
6. `OW-E8` (+E9) climate stages run **after rivers**, violating audit order ("before `triangle_values`"). Determinism of *order* is wrong even if each stage is deterministic.
7. `OW-E5` / `OW-E18` / `SC-E4` ‚Äî validation + missing-stage error are **never wired into the real `generate` path** (errors swallowed to a log line; `validate_region_rings` only called in tests). "Fails loudly in dev" is false.
8. `OW-E15` ‚Äî registry says `Scaffolded` at "stages::elevation smoothing" but **no smoothing stage exists** (`grep` finds none in `stages/`).

**P2 ‚Äî genuinely missing entries for items the audit lists (incl. some "Done"):**
9. `SC-E2` (audit: **Done**) ‚Äî no registry entry, no inward-tri gate in crate.
10. `PIPE-det` ‚Äî no `deterministic_globe` pipeline variant exists.
11. `SA-0.2/E1/E2` async job ‚Äî absent; the headline "C++ owns long-running work on worker threads" is unmodelled.
12. `OW-P1`, `GW-E7` persistence ‚Äî no serialize code.

**P3 ‚Äî honest deferrals (acceptable, noted for completeness):**
`OW-E10/E17/E19`, `GW-E5/E6/E8`, `SC-E5/E6/E7`, all of `PL/SV/GE/SP/EC/PR-0.x` ‚Äî flagged `Deferred`/Later truthfully.
