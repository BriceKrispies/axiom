## Top fidelity gaps (prioritised)

1. **Locator fast-path is not provably correct (OW-E3 / P1).** The "1-ring of a
   lat/lon grid guarantees nearest" claim is false near the poles because the grid
   is not equal-area; `locate_region` can silently return a non-nearest region and
   `sample_macro`/`sample_surface` inherit it. The single n=137 test never visits
   the failure mode. **This is a correctness bug presented as a proven invariant.**

2. **Determinism (SC-E3) has no real regression guard.** No `deterministic_globe`
   pipeline exists, `determinism.rs` has zero tests, and no golden hash is pinned.
   The only check compares two in-process runs, which proves purity, not stability
   across builds. Any change to field count/order/values silently moves the hash.

3. **OW-E18 / SC-E4 topology validation is dead code in the pipeline.** Marked
   `Implemented` in the requirement registry but never called by `generate`; there
   is no `validate_topology` stage and no abort. The acceptance criterion ("validate
   before hydrology, fail loud in dev") is unmet, and the registry overclaims it.

4. **Climate stage ordering contradicts OW-E8.** `wind_field → moisture_advection
   → rain_shadow` run *after* `triangle_values` and the river/flood stages, whereas
   OW-E8 requires moisture before `triangle_values`. Triangle/elevation fields are
   finalised against pre-climate (or flood-mutated) data. Off-spec.

5. **OW-E21 ±3% land gate is untested end-to-end.** The stage is correct in
   isolation but the only test tolerates ±5% on synthetic data; nothing runs the
   real pipeline and checks `atlas` land fraction vs the genome target at 0/40/100%
   within ±3%. The headline acceptance gate has no automated proof.

6. **"Seam Δ = 0 by construction" is overstated (SC-E8).** Adjacent chunks feed
   *different* float origins into the exp-map, so the seam is rounding-bounded, not
   bit-zero; the test tolerates 1 mm. And `sample_macro` is C0-discontinuous at
   region boundaries (the candidate set jumps), which the flat-atlas seam test
   cannot expose — a real risk to GW-E19 smoothness and the 6 m budget at seams.

7. **Several "monotonic / downwind" tests assert the weak version of the invariant.**
   priority_flood's monotonicity test accepts a single ≤-neighbour (plateaus pass
   with no real outflow path); moisture_advection's test only checks one value > 0
   and a coarse ordering; rain_shadow's test can't separate orographic gain from
   plain elevation. These are "named but not actually proving the thing."

---

## 5-line summary

1. The substrate is real and surprisingly complete — every stage does directional/algorithmic work, not stubs — and all 104 tests pass; this is genuine, not coverage theater at the unit level.
2. But the headline determinism guarantee (SC-E3) has **no golden hash, no deterministic_globe pipeline, and zero tests** in `determinism.rs`; it only proves same-process purity.
3. The locator's "fast-path == brute force" is **geometrically false at the poles** (non-equal-area lat/lon grid) and is "proven" by one benign n=137 test — a latent correctness bug.
4. OW-E18 topology validation is marked Implemented but is **dead code never called by the pipeline**, and OW-E5 "fail loud" actually fails *silent* (missing stage → flat planet, generation continues).
5. The climate chain is ordered **after** `triangle_values` (contradicting OW-E8), the OW-E21 ±3% land gate is **untested end-to-end**, and "seam Δ = 0 by construction" is overstated (rounding-bounded + C0 jumps at region boundaries the tests never cross).
