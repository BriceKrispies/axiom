| Lint name                                | Summary                                                                                                                                |   |                                      |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | - | ------------------------------------ |
| `engine_no_unsafe`                       | Bans `unsafe` code except in explicitly approved low-level crates or annotated escape hatches.                                         |   |                                      |
| `engine_no_transmute`                    | Bans `std::mem::transmute` and similar memory reinterpretation APIs.                                                                   |   |                                      |
| `engine_no_uninit_memory`                | Bans `zeroed`, `uninitialized`, and direct `MaybeUninit` usage outside approved storage primitives.                                    |   |                                      |
| `engine_no_static_mut`                   | Bans `static mut` and global mutable state.                                                                                            |   |                                      |
| `engine_no_ffi_without_boundary`         | Bans `extern`, `no_mangle`, `link`, and FFI symbols outside approved platform/FFI crates.                                              |   |                                      |
| `engine_no_time_in_sim`                  | Bans direct access to wall-clock or monotonic time inside deterministic simulation code.                                               |   |                                      |
| `engine_no_random_in_sim`                | Bans direct random number generation inside simulation code; randomness must come from explicit seeded inputs.                         |   |                                      |
| `engine_no_hashmap_in_sim`               | Bans `HashMap`/`HashSet` in deterministic simulation because iteration/order behavior can harm replayability.                          |   |                                      |
| `engine_no_direct_io`                    | Bans direct filesystem, network, process, environment, and OS access outside boundary crates.                                          |   |                                      |
| `engine_no_thread_spawn`                 | Bans direct thread/task spawning outside scheduler/platform layers.                                                                    |   |                                      |
| `engine_no_async_in_runtime`             | Bans `async`/`.await` in strict runtime, sim, and hot-path zones.                                                                      |   |                                      |
| `engine_no_panic_paths`                  | Bans `panic!`, `todo!`, `unimplemented!`, `unwrap`, and `expect` in runtime engine code.                                               |   |                                      |
| `engine_no_unwrap_expect`                | Specifically bans `unwrap()` and `expect()` unless inside tests or approved startup validation.                                        |   |                                      |
| `engine_no_todo_unimplemented`           | Bans `todo!()` and `unimplemented!()` from committed engine code.                                                                      |   |                                      |
| `engine_no_source_branching`             | Bans visible control flow like `if`, `match`, `while`, `loop`, `for`, `?`, `&&`, and `                                                 |   | ` inside branchless/primitive zones. |
| `engine_branch_budget`                   | Limits the number of branches allowed in marked hot-path or runtime functions.                                                         |   |                                      |
| `engine_no_hidden_result_branch`         | Bans the `?` operator in strict zones because it creates implicit early-return control flow.                                           |   |                                      |
| `engine_no_raw_loops`                    | Bans raw `for`, `while`, and `loop` except inside approved bounded-loop primitives or supervisor loops.                                |   |                                      |
| `engine_loop_must_have_bound`            | Requires every allowed loop to have a statically visible upper bound.                                                                  |   |                                      |
| `engine_no_unbounded_iterator_chain`     | Bans iterator chains like `map`, `filter`, `fold`, `collect`, `find`, and `any` unless using approved bounded iterators.               |   |                                      |
| `engine_no_recursion`                    | Bans direct or indirect recursive calls in runtime engine code.                                                                        |   |                                      |
| `engine_no_alloc_in_hot_path`            | Bans obvious allocation APIs like `Vec::new`, `vec![]`, `String::new`, `Box::new`, `format!`, `collect`, and `to_string` in hot paths. |   |                                      |
| `engine_no_alloc_after_init`             | Bans allocation after the engine runtime has entered its sealed/run phase.                                                             |   |                                      |
| `engine_no_vec_growth`                   | Bans growing dynamic collections like `push`, `extend`, `insert`, and `resize` on `Vec` in runtime zones.                              |   |                                      |
| `engine_require_bounded_collections`     | Requires runtime collections to use fixed-capacity/bounded storage types instead of normal dynamic containers.                         |   |                                      |
| `engine_no_dynamic_dispatch_in_hot_path` | Bans `dyn Trait`, `Box<dyn Trait>`, `&dyn Trait`, and similar dynamic dispatch in hot paths.                                           |   |                                      |
| `engine_no_runtime_type_branch`          | Bans `Any`, `TypeId`, `downcast_ref`, and `downcast_mut` as runtime type escape hatches.                                               |   |                                      |
| `engine_no_trait_objects_in_components`  | Bans ECS components from storing trait objects or service-like runtime handles.                                                        |   |                                      |
| `engine_components_are_data_only`        | Requires ECS components to be plain data with no behavior-heavy methods, IO handles, locks, or async state.                            |   |                                      |
| `engine_no_component_update_methods`     | Bans component-local `update()` style behavior so logic stays in systems.                                                              |   |                                      |
| `engine_systems_use_command_buffer`      | Prevents systems from directly creating/destroying entities except through an approved command buffer.                                 |   |                                      |
| `engine_no_forbidden_imports`            | Enforces crate/layer boundaries, such as sim not importing render or renderer not importing gameplay.                                  |   |                                      |
| `engine_no_layer_skipping`               | Prevents modules from importing across architectural layers instead of going through the allowed adjacent boundary.                    |   |                                      |
| `engine_no_renderer_gameplay_coupling`   | Bans renderer code from referencing gameplay concepts like `Player`, `Enemy`, `Health`, or `Weapon`.                                   |   |                                      |
| `engine_no_gameplay_in_platform`         | Bans platform/window/OS adapter code from knowing about game-specific simulation concepts.                                             |   |                                      |
| `engine_no_public_api_without_marker`    | Bans accidental `pub` APIs unless marked with something like `#[engine_api]`.                                                          |   |                                      |
| `engine_no_pub_fields`                   | Bans public struct fields unless the type is explicitly marked as a public data contract.                                              |   |                                      |
| `engine_no_wildcard_imports`             | Bans `use foo::*` because it makes agent-written code less greppable and less reviewable.                                              |   |                                      |
| `engine_no_large_files`                  | Fails files/modules over a configured line-count threshold.                                                                            |   |                                      |
| `engine_no_large_functions`              | Fails functions over a configured size or complexity threshold.                                                                        |   |                                      |
| `engine_no_large_structs`                | Fails structs with too many fields unless explicitly approved.                                                                         |   |                                      |
| `engine_no_large_enums`                  | Fails enums with too many variants unless explicitly approved.                                                                         |   |                                      |
| `engine_no_large_impl_blocks`            | Fails oversized `impl` blocks that suggest god-object behavior.                                                                        |   |                                      |
| `engine_require_module_docs`             | Requires public modules to explain their role and allowed dependencies.                                                                |   |                                      |
| `engine_require_escape_hatch_reason`     | Requires every `#[engine_escape_hatch]` to include a non-empty reason.                                                                 |   |                                      |
| `engine_escape_hatch_scope_limit`        | Restricts escape hatches to the smallest possible item instead of allowing whole-crate exceptions.                                     |   |                                      |
| `engine_no_unitless_float_public_api`    | Bans naked `f32`/`f64` in public APIs where unit wrappers like `Meters`, `Seconds`, or `Radians` should be used.                       |   |                                      |
| `engine_no_float_in_deterministic_sim`   | Bans or restricts floating-point math in deterministic simulation unless wrapped/approved.                                             |   |                                      |
| `engine_no_float_trig_in_sim`            | Bans `sin`, `cos`, `tan`, and similar platform-sensitive math in deterministic simulation.                                             |   |                                      |
| `engine_no_indexing_in_strict_code`      | Bans unchecked-looking indexing syntax like `items[i]` in strict code unless using approved bounded accessors.                         |   |                                      |
| `engine_no_logging_in_hot_path`          | Bans logging macros in frame/update/render hot paths.                                                                                  |   |                                      |
| `engine_no_locks_in_hot_path`            | Bans `Mutex`, `RwLock`, blocking waits, and lock acquisition in hot paths.                                                             |   |                                      |
| `engine_no_blocking_calls`               | Bans known blocking operations inside runtime tick/update paths.                                                                       |   |                                      |
| `engine_no_format_in_runtime`            | Bans `format!` and runtime string construction in strict runtime zones.                                                                |   |                                      |
| `engine_no_stringly_typed_ids`           | Bans string IDs for engine resources/entities where typed handles should be used.                                                      |   |                                      |
| `engine_no_schema_less_packets`          | Bans loosely typed packet structures, JSON packet payloads, or dynamic field maps in runtime networking.                               |   |                                      |
| `engine_packet_layout_is_bounded`        | Requires packets to have fixed or explicitly bounded encoded sizes.                                                                    |   |                                      |
| `engine_no_direct_network_mutation`      | Prevents network receive code from directly mutating simulation state instead of producing commands/events.                            |   |                                      |
| `engine_supervisor_loop_only`            | Allows infinite loops only in explicitly marked supervisor functions, never inside per-frame/tick work.                                |   |                                      |
| `engine_tick_work_is_bounded`            | Requires all work performed inside one tick/frame to have declared maximum budgets.                                                    |   |                                      |
| `engine_no_dependency_escape`            | Bans direct use of unapproved crates from strict engine modules.                                                                       |   |                                      |
| `engine_dependency_must_be_wrapped`      | Requires external dependencies to be accessed through approved engine wrapper crates.                                                  |   |                                      |
| `engine_no_clone_large_types`            | Bans `clone()` on known large engine types in hot paths.                                                                               |   |                                      |
| `engine_no_collect_runtime`              | Specifically bans `collect()` in runtime code because it hides iteration and allocation.                                               |   |                                      |
| `engine_no_drain_unbounded`              | Bans `drain(..)` or unbounded queue draining in one tick/frame.                                                                        |   |                                      |
| `engine_queue_has_per_tick_budget`       | Requires event/command/message queues to expose and enforce a max processed count per tick.                                            |   |                                      |
| `engine_state_machine_zone_required`     | Requires branching-heavy transition logic to live in marked state-machine/decision modules.                                            |   |                                      |
| `engine_no_ad_hoc_state_branching`       | Bans random local `match state` / `if state ==` logic outside approved state-machine zones.                                            |   |                                      |
